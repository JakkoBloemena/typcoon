## 3. Draft write-assignments (ids TBD, for the dispatcher)

Each is one developer-sized task: **add one article object to
`scripts/content/nl.mjs`'s `articles` array in the existing shape**
(`slug, title, description, h1, date, updated, readMin, lead, sections[{h2,html}], faq[{q,a}]`),
then `npm run build` (which runs `scripts/gen-content.mjs`) to generate the page at
`/blog/<slug>/`. The generator already produces Article + BreadcrumbList schema, hreflang,
the blog-index card, the sitemap entry, and the shared "Speel gratis" CTA box — so those are
**verify-the-output** criteria, not author-them criteria. What the author owns is the copy,
the target-term placement, and the internal links inside the section HTML.

**Shared acceptance criteria (apply to A–D):**

- [ ] New article object added to `scripts/content/nl.mjs` `articles` array, matching the
      existing field shape exactly; `npm run build` completes clean and emits
      `/blog/<slug>/` (verify the file is generated).
- [ ] Target term appears in the `title`, the `h1`, the `slug`, and within the first ~100
      words (the `lead`) — once each, naturally. **No repetition beyond natural use; a
      reader must never feel the keyword was forced (guardrail 7 / SEO.md §9).**
- [ ] Body is **800–1500 words** of substantive, accurate help; every product claim matches
      what Typcoon actually does (adaptive engine, accuracy multiplier, free base tier,
      no ads, no account required, optional adult-only account) — no over-claiming.
- [ ] Contains an internal link to the pillar `/leren-typen-voor-kinderen/` **and** at least
      one link to a relevant sibling spoke (see per-article notes).
- [ ] Generated page shows valid Article + BreadcrumbList JSON-LD and appears in
      `sitemap.xml` (verify generator output — no hand-editing of schema/sitemap).
- [ ] `npm test` green.
- [ ] Reviewed against guardrail 7: no keyword-stuffing, no thin/spun content, no invented
      claims, no disparagement of competitors — the page earns its ranking by being useful.

## 4. How measurement (006) re-ranks future batches — without blocking this one

Assignment 006 wired a first-party, cookieless funnel endpoint (`api/track.js` →
`events` table: `pageview → game_start → engaged_session → parent_opt_in`), with a weekly
readout. **Google Search Console is not yet verified** (006 flagged it as needing the
Shareholder's access) — so query/impression/CTR data does not exist yet. That is *why*
batch 2 is chosen from SEO.md's existing intent research and is **not** blocked on 006.

Once 006's two data sources are live, batch 3 selection changes from *intent-guess* to
*measured value*, in this order of authority:

1. **Search Console (queries/impressions/CTR/position)** — the primary re-ranker.
   - **Promote near-misses over net-new:** any existing page sitting at position 5–15 with
     real impressions is a cheaper win (better title/depth/internal links) than a brand-new
     article — SEO.md §7's explicit guidance. Batch 3 should budget for these refreshes
     *before* new spokes.
   - **Validate/kill the guesses in this doc:** if `nitro type` or `typecursus kind` show
     negligible NL impressions, drop them; if an unforeseen query surfaces with volume,
     it jumps the queue. **No article gets written on an assumed volume once real
     impression data exists.**
2. **First-party funnel (`events` table)** — the value re-ranker.
   - Rank published articles by **game-starts per session** and, downstream, **opt-ins**
     (the charter's proxy metric), not by pageviews. An article with traffic but no
     game-starts is mis-targeted (wrong intent or weak CTA) and gets *fixed*, not
     multiplied; a topic that reliably drives game-starts earns *more* articles around it.
   - This makes the charter's funnel (`visit → game-start → engaged → opt-in`) the tie-break
     between two candidate topics of similar search intent.
3. **Re-scoring cadence:** re-run this gap map at batch 3 with the 006 data attached; keep
   the "decided not to build" list honest (drop items measurement disproves, add items it
   reveals). The deferred `blind-typen-zonder-kijken` refresh and the held `typecursus kind`
   page are the first two candidates to re-evaluate against real numbers.

Until then: ship batch 2 on intent; let 006 grade it in retrospect and steer batch 3.
